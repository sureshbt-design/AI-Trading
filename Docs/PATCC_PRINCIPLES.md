# PATCC Principles

## Mission

PATCC is a personal financial operating system supporting investment decisions,
retirement planning, tax efficiency, household operations, and family continuity.

## Mandatory Operating Principles

1. The Morning Report must remain operational and usable before every market open.

2. Every development session must end with:
   - valid Python syntax,
   - working reports,
   - tested changes,
   - committed code,
   - a clean Git working tree.

3. Every recommendation must be:
   - evidence-based,
   - validated,
   - explainable,
   - associated with a defined timeframe.

4. PATCC must never invent probabilities, targets, account balances, tax limits,
   or financial facts when reliable information is unavailable.

5. Knowledge, events, and generated intelligence must remain separate:
   - Knowledge describes the household and its policies.
   - Events record what happened.
   - Intelligence contains regenerated analysis and recommendations.

6. Historical transactions and financial events must not be silently deleted
   or overwritten.

7. PATCC must not store:
   - passwords,
   - PINs,
   - MFA secrets,
   - private keys,
   - cryptocurrency seed phrases.

8. Family continuity is a core system requirement.

9. Every module must provide practical value and integrate into a report,
   dashboard, ledger, alert, or decision workflow.

10. If an output would cause a reasonable investor to question its logic,
    PATCC must detect and flag the issue before presenting it as actionable.

## Development Standard

Every sprint must leave PATCC in a stable, reusable, recoverable state.
